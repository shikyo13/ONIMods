[h1]Duplicant Status Bar — Changelog[/h1]

[h2]v2.5.1 — Game UI Scale Support[/h2]
[list]
[*] The status bar now scales with the in-game UI Scale setting (Options → Graphics) — no longer stuck at 100% when the rest of the UI is scaled up
[*] Scale updates live when the setting changes, no restart needed
[*] Drag-to-resize and all other features unaffected
[*] Fixed NRE in filter button construction (Image added before RectTransform access)
[/list]

[h2]v2.5.0 — Calorie Tracking, Sort by Calories, Color Centralization[/h2]
[list]
[*] Calorie percentage now shown in hover tooltip with green-to-red color gradient (60/40/20% breakpoints)
[*] Body temperature in tooltip is now color-coded by deviation from normal (36.85°C) — green within ±1.5°, amber ±3.5°, orange ±5.5°, red beyond
[*] New sort mode: "Calories (lowest first)" — hungriest dupes appear first in the bar, useful for Carnivore achievement runs
[*] All mod colors centralized into ColorUtil.cs — single source of truth for stat gradients, alert badges, stress tiers, and UI chrome; enables future theming
[*] Mod settings "Homepage" button now links to the Steam Workshop page
[*] Assembly version bumped to 2.5.0
[*] Added German, Spanish, and French translations (now 7 languages total)
[/list]

[h2]v2.4.0 — Localization (7 Languages)[/h2]
[list]
[*] All UI text is now localizable — tooltip labels, alert names, option labels/descriptions, and the bar header
[*] Translations included for: Chinese, Korean, Russian, and Japanese (matching ONI's supported languages)
[*] Translations load automatically when the game language matches a supported locale
[*] English remains the default; community translation contributions welcome via .po files in the translations/ folder
[/list]

[h2]v2.3.4 — Portrait Tuning + Code Quality[/h2]
[list]
[*] Tuned portrait feature offsets for better eye/mouth/hat alignment across all dupe variants
[*] Added 3° counter-clockwise eye rotation to level out ONI's 3/4 perspective tilt
[*] Unified alert color palette — overlays, badges, and tooltips now share a single color source
[*] Tooltip alert text is now full-brightness static color instead of pulsing alpha (much easier to read); overjoyed rainbow animation unchanged
[*] Fixed sprite cache key collision that could cause wrong expression frames on rare hash overlaps
[*] Batch-read source pixels in portrait compositing for fewer managed-to-native calls
[*] Added cache eviction for dead/transferred dupes (previously leaked portrait textures)
[*] Diagnostic dump disabled by default (was running once per session in production)
[*] Sparkle expression debug log now fires once instead of every 0.25s per overjoyed dupe
[*] Reusable HashSet for dupe ID tracking (was allocating a new one every 0.25s)
[*] Named constants for all portrait positioning offsets
[/list]

[h2]v2.3.3 — Edge-Anchored Feature Positioning[/h2]
[list]
[*] Eyes and mouth now use edge-anchored positioning — eye bottom edge and mouth top edge are fixed at absolute y-coordinates, so the gap between features is constant regardless of sprite dimensions
[*] Fixes per-dupe eye-mouth spacing variation (e.g., Hassan's mouth crowding his eyes while other dupes looked correct)
[/list]

[h2]v2.3.2 — Y-Offset Tuning + Sparkle Eye Fix[/h2]
[list]
[*] Tuned eye and mouth Y-offsets for better vertical placement on head
[*] Fixed Sparkle expression showing white circle overlay instead of normal eyes — now uses known-good frame indices
[/list]

[h2]v2.3.1 — Expression Sprite Sizing + Blink Fallback[/h2]
[list]
[*] Proportional size clamping for expression sprites — oversized frames (e.g., Sparkle mouth) are uniformly scaled down to prevent clipping
[*] Added Tired face as third blink fallback when Sleep and Dead faces aren't available in kanim variants
[/list]

[h2]v2.3.0 — Expression-Driven Portraits[/h2]
[list]
[*] Portraits now show contextual facial expressions based on alert state and stress level
[*] 13 expression types: Neutral, Happy, Angry, Suffocate, Cold, Hot, Hungry, Tired, Sick, Sparkle, Uncomfortable, Dead, Productive
[*] Blink system with randomized per-dupe timing (3-8s interval, 0.15s blink duration)
[*] Expression frames discovered at runtime from head_master_swap_kanim — works with all body type variants
[*] New EnableExpressions option (default on) to revert to static portraits if desired
[/list]

[h2]v2.2.0 — Per-Alert Visual Effects[/h2]
[list]
[*] Animated tooltip text: each alert gets per-character color animation with glow
[*] Overjoyed alerts use smooth rainbow color cycling; other alerts pulse monochromatically
[*] New AlertEffects system with three animation patterns (Pulse, Heartbeat, Flicker)
[*] Incapacitated alert: detects dupes bleeding out (120s to death)
[/list]

[h2]v2.1.3 — Pause-Safe Timers + Distinct Alert Colors[/h2]
[list]
[*] Idle and stuck timers no longer tick while the game is paused — no more false alerts from reading tooltips
[*] Redesigned alert color palette: 12 alerts now span the full color wheel instead of clustering in orange/amber, making badges instantly distinguishable at a glance
[/list]

[h2]v2.1.2 — Stuck & Idle Detection[/h2]
[list]
[*] Stuck alert: detects dupes who can't path to the Printing Pod for 10+ seconds
[*] Idle alert: fires when a dupe has been idle for 30+ seconds
[*] Both alerts configurable via mod options with per-alert toggles
[/list]

[h2]v2.1.1 — UI Hardening[/h2]
[list]
[*] Panel position clamped to screen on load and after dragging — no more stuck-off-screen panels after resolution changes
[*] Drag now uses Unity EventSystem interfaces instead of raw Input polling — no more conflicts with building placement or other game clicks
[*] Tooltip clamped to screen bounds — stays visible when hovering dupes near edges
[*] Batched circle sprite generation (SetPixels32) and reused StringBuilder for less GC pressure
[*] Removed redundant horizontal ContentSizeFitter that fought anchor-based stretching
[/list]

[h2]v2.1 — Visual Overhaul + Health[/h2]
[list]
[*] Complete visual overhaul: ONI-native color palette, rounded corners via procedural 9-slice sprites, game font (GRAYSTROKE REGULAR SDF)
[*] Burgundy header with bevel highlights, panel Mask for rounded clipping
[*] Health fill bar — anchor-based vertical drain with 3-segment color gradient (green → yellow → orange → red), hidden at full HP
[*] Dark-red damage overlay above portrait proportional to missing HP
[*] Multi-badge alert system — up to 3 simultaneous badges with adaptive sizing and per-alert hold timers
[*] Bitmask-based AlertMask on snapshots for multiple concurrent alerts
[*] Tooltip lists all active conditions with color-coded entries
[*] Multi-row wrapping with configurable MaxBarWidth (default 50% screen) and MaxBarRows (default 3)
[*] ScrollRect with thin rounded scrollbar when rows overflow
[*] Minimum portrait size lowered to 16px for large colonies
[*] Live opacity update, portrait size change detection without restart
[*] Scalding/hypothermia alerts now use ScaldingMonitor SMI instead of body temperature
[*] Auto-deploy MSBuild target
[/list]

[h2]v1.2 — Grid Layout + Resize[/h2]
[list]
[*] GridLayoutGroup replaces HorizontalLayoutGroup — rows wrap automatically
[*] Drag-to-resize handle at bottom-right corner (24–96px range, persisted)
[*] MaxDupesPerRow config option (0 = auto)
[*] DisplayMode option: Portraits / Initials (updates live, no restart)
[/list]

[h2]v1.1 — Stability + New Alerts[/h2]

[*] Sort damping: 3s timer + 5% dead-zone to prevent jitter
[*] Size stability: skip recompute on same dupe count
[*] New alerts: Irradiated, Starving, Bladder Urgent (with config toggles)
[*] Tooltip: bladder%, starving status, idle fallback
[*] Badge hysteresis (0/1/3s hold times by urgency)
[*] Exponential color smoothing
[*] Dirty-check on TMP text to avoid unnecessary rebuilds
[*] Colorblind fix: Mild tier color adjusted
[/list]
