[h1]Duplicant Status Bar - Changelog[/h1]

[h2]v2.8.4 - Position & Localization Fixes[/h2]
[list]
[*] [b]Fixed header text overlap with translations[/b] - filter button and title now measure translated text width at runtime instead of hardcoded pixel widths. German, Russian, Japanese and other languages no longer overlap in the header (GitHub #14)
[*] [b]Fixed unicode escape in translations[/b] - arrow character was displaying as literal "\u25BC" text instead of ▼ in non-English languages
[*] [b]Position recovery on resolution change[/b] - bar position now remaps proportionally when screen resolution or UI scale changes between sessions, with a deferred safety clamp after first layout pass
[/list]

[h2]v2.8.3 - Mod Updater Restart Loop Fix[/h2]
[list]
[*] [b]Fixed endless restart loop with Mod Updater[/b] - PLib was writing config.json into the mod's Steam Workshop folder, which could trigger Mod Updater's change detection on every launch. Config now stored in PLib's shared config folder (GitHub #13)
[*] [b]Fixed AssemblyVersion mismatch[/b] - DLL version was stuck at 2.7.0.0 since v2.7, now properly synced to 2.8.3. This caused version tracking confusion with Mod Updater
[*] [b]Fixed portraits showing initials on one asteroid[/b] - when the bar was resized small with many dupes, card size dropped below the portrait threshold. Threshold lowered from 36px to 20px so portraits display at smaller sizes (GitHub #10)
[*] [b]Crash-safe initialization[/b] - all Harmony patches and mod loading now wrapped in try-catch so DSB can never crash the game during startup
[*] [b]Version diagnostic[/b] - Player.log now always shows [DSB] version and load path for easier troubleshooting
[*] [b]Build cleanup[/b] - deploy no longer includes stale config.json, .pdb files, or unused funnel.png
[/list]

[h2]v2.8.2 - Management Screen Fix + Resize Improvements[/h2]
[list]
[*] [b]Fixed bar rendering on top of management screens[/b] - the status bar now hides when Research, Skills, Starmap, or other full-screen management panels are open
[*] [b]Resize system rework (beta)[/b] - the resize grip now supports independent horizontal and vertical scaling. Drag right to add/remove columns, drag down to add/remove rows, or drag diagonally to constrain both axes. Panel grows from its top-left corner with screen bounds clamping. Cards reflow smoothly per-frame during drag. [i]Note: the resize feature has been significantly reworked and may have rough edges. Please report any issues on the Workshop or Discord.[/i]
[*] [b]Fixed portrait crash[/b] - guards against null/default frame data in portrait compositing that could cause NRE with certain dupe accessory configurations (GitHub #10)
[/list]

[h2]v2.8.0 - Bionic Dupe Support + Troubleshooting Tools[/h2]
[list]
[*] [b]Bionic dupe support[/b] — Battery, Gunk, and Gear Oil stats now shown in tooltips instead of Calories/Bladder for bionic dupes
[*] [b]3 new bionic alerts[/b] — Low Battery (1 or fewer electrobanks), Low Gear Oil (<20%), Grinding Gears (no oil) with distinct colors, effects, and facial expressions
[*] [b]Calories sort handles bionics[/b] — "Calories (lowest first)" sort uses battery % for bionic dupes
[*] [b]Role filter fix[/b] — hiding a role now correctly hides all skill tiers that share the same display name
[*] [b]Per-dupe component cache[/b] — eliminates redundant GetComponent/GetSMI calls per tick for better performance
[*] [b]Reset Bar Position & Filters button[/b] — new button in mod options (General) resets the bar to top-center, un-collapses it, and clears all filters. Fixes invisible bar caused by off-screen position or stale filter state
[*] [b]Improved debug logging[/b] — when Debug Logging is enabled, dsb_debug.log now captures the full lifecycle: Harmony patch firing, UI construction, canvas position/scale, filter state with dupe names, and per-dupe filter drop reasons
[/list]

[h2]v2.7.0 — Unity 6 / Testing Branch Compatibility[/h2]
[list]
[*] [b]Fixed crash when opening Sort/Filter popup on testing branch[/b] — vanilla KScreenManager dispatches input to a screen referencing a removed TMP method; popup now calls DisableInput while open
[*] [b]Fixed ForceMeshUpdate crash on Unity 6[/b] — parameterless overload removed in built-in TMP; uses reflection to find the correct signature at runtime
[*] [b]Fixed missing checkbox glyphs on testing branch[/b] — replaced ✓/✗ (U+2713/U+2717) with [x]/[ ] since the game font lacks those codepoints on Unity 6
[*] [b]Fixed build against Unity 6 game DLLs[/b] — ImageConversionModule now references netstandard 2.1 which causes an unsuppressible CS1705; LoadImage and EncodeToPNG called via reflection instead
[*] Fixed oversized status bar on Mac Retina / high-DPI displays (GitHub issue #1)
[*] Sort/Filter button now hides when the bar is collapsed
[*] Backwards compatible — single binary works on both public and testing branches
[/list]

[h2]v2.6.0 — Extensibility API[/h2]
[list]
[*] [b]Mod API[/b] — other mods can now register custom alerts, hook into tooltips, and listen to status bar events via `DuplicantStatusBar.API.Experimental.DSBApi`
[*] Custom alerts get badge icons, tooltip entries, and animation patterns — same visual treatment as built-in alerts
[*] Tooltip hooks let external mods append custom stat lines to the hover tooltip
[*] Event system: react to alert changes, widget lifecycle, snapshot updates, and bar visibility changes
[*] All external callbacks are error-isolated — a broken mod never crashes the status bar
[*] See `docs/api-guide.md` for the full guide and examples
[/list]

[h2]v2.5.1 — In-Game Sort/Filter Popup + Game UI Scale[/h2]
[list]
[*] [b]In-game Sort/Filter popup[/b] — click the ► Sort/Filter button on the header bar to open a drop-down right below the bar. No more digging through mod settings to change sort order or hide dupes
[*] Sort modes: Stress (highest first), Calories (lowest first), Alphabetical, Job Role
[*] Smart filters: "Alerts only" shows only dupes with active alerts; "Stressed only" shows dupes above calm threshold
[*] Role filter: toggle visibility per job role (e.g. hide all Researchers); All/None buttons for quick toggling
[*] Per-dupe filter: hide/show individual dupes by name; All/None buttons
[*] All filter state persisted across sessions
[*] Camera controls disabled while popup is open — no accidental scrolling/panning
[*] [b]Respects game UI scale[/b] — the status bar now scales with the in-game UI Scale setting (Options → Graphics), updates live when the setting changes
[*] Assembly version now visible in mod options
[*] Sort/filter popup fully localized — button text, section headers, and All/None links translated into all 7 supported languages
[*] Fixed NRE in filter button construction
[/list]

[h2]v2.5.0 — Calorie Tracking, Sort by Calories, Color Centralization[/h2]
[list]

[*] Added German, Spanish, and French translations (now 7 languages total)
[/list]

[h2]v2.3.4 — Portrait Tuning + Code Quality[/h2]
[list]
[*] Tuned portrait feature offsets for better eye/mouth/hat alignment across all dupe variants
[*] Added 3° counter-clockwise eye rotation to level out ONI's 3/4 perspective tilt
[*] Unified alert color palette — overlays, badges, and tooltips now share a single color source
[*] Tooltip alert text is now full-brightness static color instead of pulsing alpha (much easier to read); overjoyed rainbow animation unchanged
[*] Fixed sprite cache key collision that could cause wrong expression frames on rare hash overlaps
[*] Batch-read source pixels in portrait compositing for fewer managed-to-native calls
[*] Added cache eviction for dead/transferred dupes (previously leaked portrait textures)
[*] Diagnostic dump disabled by default (was running once per session in production)
[*] Sparkle expression debug log now fires once instead of every 0.25s per overjoyed dupe
[*] Reusable HashSet for dupe ID tracking (was allocating a new one every 0.25s)
[*] Named constants for all portrait positioning offsets
[/list]
